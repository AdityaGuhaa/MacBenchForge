# Security Policy

## Supported Versions

Currently, only the latest major release of MacBenchForge is actively supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| 2.1.x   | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

Security is highly important to the MacBenchForge project. If you discover a security vulnerability within MacBenchForge, please **do not open a public issue**. 

Instead, please report it privately via one of the following methods:

1. **GitHub Private Vulnerability Reporting:** Navigate to the "Security" tab of this repository, click "Advisories", and select "Report a vulnerability".
2. **Direct Contact:** If you are unable to use the GitHub reporting tool, you can reach out privately via email or direct message to the repository owner (Aditya Guha).

### What to include in your report:
* A detailed description of the vulnerability and its potential impact.
* Step-by-step instructions to reproduce the issue.
* Any relevant environment details (macOS version, MacBenchForge version, etc.).

### Response Timeline
We aim to acknowledge receipt of vulnerability reports within 48 hours and will keep you informed of our progress towards a fix and public disclosure.

Thank you for helping keep MacBenchForge secure!
